# sizhu-programming 四柱编程

> 当老天是 Python 程序员

这是一个尝试用程序员思维重新理解、拆解和表达四柱命理的项目。

它的核心目标，不是把传统术数神秘化，而是把其中**可以结构化、规则化、程序化**的部分，尽量用现代软件工程语言重新描述出来：  
把“经验判断”拆成“数据结构、规则系统、分析流程、解释层”。

当前仓库包含两部分内容：

1. **原始长文档**：`sizhu_programming.md`  
   这是本项目最早的主体内容，集中整理了“四柱如何程序化表达”的整体思路、章节结构和分析框架。

2. **最小可运行原型代码**：`sizhu/` + `tests/`  
   这是在文档基础上迈出的第一步，用最小规模的 Python 代码实现一部分确定性最强的底层规则。

---

## 项目定位

这个项目不是“算命网站源码”，也不是“自动断命神器”。

它更像一个持续建设中的实验：

- 一方面，尝试把四柱系统中可编码的部分抽象出来；
- 另一方面，也承认很多高阶判断仍然属于解释层，而不是简单几条规则就能完全替代的人类经验。

所以，本项目的目标不是一步到位，而是分层推进：

1. **数据层**：天干、地支、五行、阴阳、十神、藏干等基础对象  
2. **规则层**：映射关系、查表逻辑、基础推导  
3. **分析层**：组合分析、结构判断、阶段性分析  
4. **解释层**：把规则结果组织成更接近人类阅读的说明文本

当前版本主要完成的是前两层，并开始搭建第三层的基础骨架。

### 文件说明

  项目的原始长文档，保留最完整的思路、章节设计和理论框架。

- `pyproject.toml`  
  Python 项目的基础配置文件。

  基础测试文件，用来验证核心规则是否符合预期。

## 快速开始

### 1. 克隆仓库

```bash
git clone https://github.com/gongzisahan/sizhu-programming.git
cd sizhu-programming
```

### 2. 安装

```bash
pip install -e .
```

### 3. 运行示例

```bash
python -m sizhu.cli --year 甲子 --month 丙寅 --day 丁酉 --hour 庚戌
```

你会得到一份 JSON 格式的基础分析结果，包括：

- 四柱
- 日主
- 天干十神
- 地支藏干十神
- 明面五行统计

### 4. 运行测试

```bash
python -m unittest
```

## 这个项目现在能做什么

当前版本适合做这些事：

- 把“四柱编程”这件事说清楚
- 给后来继续开发的人提供方向
- 验证最底层规则能不能写成代码
- 作为后续大运、流年、格局、解释层的基础

当前版本还**不适合**这些期待：

- 直接替代完整人工命理分析
- 一键生成高质量断语
- 处理全部流派分歧
- 保证所有边界情形都已覆盖

## 开发原则

这个项目后续会尽量遵循以下原则：

### 1. 先做确定性强的部分

优先实现那些可以清楚定义输入、输出和规则的内容。  
例如：十神、藏干、五行映射、基础推导。

### 2. 把“学派差异”与“底层规则”分开

底层规则尽量保持稳定；  
不同流派的解释方式，未来更适合作为不同策略层或插件层来实现。

### 3. 先保证可验证，再追求复杂

先写得对，再写得多。  
先有测试，再谈扩展。

### 4. 文档与代码并行推进

这个项目的价值不只在代码，也在于把“如何把四柱程序化”讲清楚。  
所以文档不会消失，而会继续作为设计说明存在。
